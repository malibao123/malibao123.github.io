3.基于bs4库的HTML遍历方法

1.标签书的下行遍历

  from bs4 import BeautifulSoup
  import requests
  url = "https://python123.io/ws/demo.html"
  response = requests.get(url)
  demo = response.text
  soup = BeautifulSoup(demo,'html.parser')
  soup.body.contents #会列举出body的所有子标签
  ################ 遍历body的所有子标签############3
  for child in soup.body.children:
      print(child)
  ################ 遍历body的所有子孙标签############3
  for descendant in soup.body.descendants:
      print(descendant)

4.信息标记形式: XML，JSON，YAML

1.XML:扩展标记语言,类似于HTML

  <person>
      <firstName>Tian</firstName>
      <lastName>Song</lastName>
      <address>
          <streetAddr>中关村南大街5号</streetAddr>
          <city>北京市</city>
          <zipcode>100081</zipcode>
      </address>
      <prof>Computer System</prof>
       <prof>Security</prof>
  </person>

2.JSON:JS 对象简谱,以键值对的形式存在；例如，当一个键对应多个值，多个 值用列表包起来

  {
      "firstName" : "Tian",
      "lastName" : "Song" ,
      "address" : {
          "streetAddr" : "中关村南大街5号" ,
          "city" : "北京市" ,
          "zipcode" : "100081"
          } ,
      "prof" : [ "Computer System" , "Security" ]
  }

3.YAML ：类似于python语法格式，用缩进表示所属关系，只能用空格，不能tab

  firstName : Tian
  lastName : Song
  address :
   streetAddr : 中关村南大街5号
   city : 北京市
   zipcode : 100081
  prof :
   ‐Computer System
   ‐Security

4.三种标记形式的比较

XML 最早的通用信息标记语言，可扩展性号，但是标记符号多，内容繁琐 适用于internet上信息交互与传递
JSON 信息有类型，适合程序处理(js),较XML简洁 移动应用云端和节点的信息通信
YAML 信息无类型，文本信息比例高，可读性强 各类系统的配置文件，有注释易读

5.信息提取

方法一：完整解析信息的标记形式，再提取关键信息。XML，JSON，YAML；需要标记解析器，如bs4库表签树的遍历。

​ 优：信息解析准确

​ 缺：提取过程繁琐，速度慢

方法二：无视标记形式，直接进行搜索；例如word中文本搜索函数

​ 优：提取过程快，速度快

​ 缺：提取信息准确性差

方法三：一、二结合，需要标记解析器和文本查找函数

​ <>.find_all(name,attrs,recursive,string,**kwarg) 返回一个列表类型，存储查找结果

  • name 对标签名称的检索字符串；

    find_all('a'), find_all(['a','p'])

  • attrs 对标签属性值的检索字符串，对属性进行检索

    find_all

    find_all('a',id='id1')

  • recursive: 是否对子孙全部检索，默认为True

  • string: <>…</> 中字符串区域进行检索
